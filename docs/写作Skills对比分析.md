## 🎯 详细功能对比

### 1️⃣ copywriting (coreyhaines31/marketingskills)

**📌 核心定位：** 专业营销文案写作专家

**支持平台：**
- 网站（Homepage、Landing Page、Pricing Page、Feature Page、About Page）
- 产品页面
- 落地页
- 营销材料

**去 AI 味能力：⭐⭐⭐⭐⭐**

专业反 AI 味规则：
```markdown
✅ 必须遵循的原则：
1. Simple over complex（"Use" not "utilize"）
2. Specific over vague（避免 "streamline", "optimize", "innovative"）
3. Active over passive（主动语态）
4. Confident over qualified（移除 "almost", "very", "really"）
5. Show over tell（描述结果，不用副词）
6. Honest over sensational（绝不编造数据）

❌ 快速质量检查清单：
- 有没有外行听不懂的行话？
- 句子是否想表达太多内容？
- 是否使用被动语态？
- 有没有感叹号？（删除它们）
- 有没有缺乏实质的营销术语？
```

**特色功能：**
- 客户语言匹配（使用客户真实用词）
- 页面结构框架（完整的 Above the Fold 指导）
- CTA 文案公式
- 配套 copy-editing skill 用于深度润色

**自动发布：** ❌ 不支持

**最佳使用场景：**
- 网站营销文案
- 产品描述
- 销售页面
- 需要转化率优化的文案

### 2️⃣ social-content (coreyhaines31/marketingskills)

**📌 核心定位：** 社交媒体内容策略专家

**支持平台：**
| 平台 | 发布频率 | 核心格式 | 支持度 |
|------|---------|---------|--------|
| **LinkedIn** | 3-5次/周 | Carousels, 故事贴 | ⭐⭐⭐⭐⭐ |
| **Twitter/X** | 3-10次/天 | Threads, 热点观点 | ⭐⭐⭐⭐⭐ |
| **Instagram** | 1-2帖+Stories/天 | Reels, Carousels | ⭐⭐⭐⭐ |
| **TikTok** | 1-4次/天 | 短视频 | ⭐⭐⭐ |
| **Facebook** | 1-2次/天 | 社群、原生视频 | ⭐⭐⭐ |

**去 AI 味能力：⭐⭐⭐⭐**

专业内容公式：
```markdown
📝 Hook 公式（避免 AI 味）：
1. Curiosity Hooks（好奇心）
   - "I was wrong about [common belief]."
   - "[Impressive result] — and it only took [surprisingly short time]."

2. Story Hooks（故事）
   - "Last week, [unexpected thing] happened."
   - "3 years ago, I [past state]. Today, [current state]."

3. Contrarian Hooks（反常识）
   - "Unpopular opinion: [bold statement]"
   - "[Common advice] is wrong. Here's why:"

🎨 内容支柱系统（避免千篇一律）：
- 行业洞察（30%）
- 幕后故事（25%）
- 教育内容（25%）
- 个人故事（15%）
- 推广内容（5%）
```

**特色功能：**
- 内容日历模板
- 内容复用系统（1篇博客 → 多平台内容）
- 参与度优化策略
- 逆向工程病毒内容框架

**自动发布：** ❌ 不支持（提供排期建议）

**最佳使用场景：**
- 个人品牌建设
- B2B 思想领导力
- 社交媒体内容规划
- 多平台内容复用

## ❌ 自动发布现状

### 为什么所有 Skills 都不支持自动发布？

**技术原因：**
1. **API 限制** - 大部分平台不提供公开 API
   - 公众号：需要认证服务号 + 开发
   - 小红书：无公开 API
   - 知乎：无公开 API
   - 微博：API 限制严格

2. **安全考虑** - 自动发布需要账号授权
3. **内容审核** - 平台需要人工审核

**现有替代方案：**

```markdown
半自动发布工作流：

方案 1：复制粘贴（最简单）
1. Skill 生成内容 → 保存到本地文件
2. 手动复制到平台编辑器
3. 调整排版和图片
4. 预览和发布

方案 2：浏览器自动化（需额外配置）
1. 使用 agent-browser skill
2. 自动填写表单和上传
3. 仍需人工确认发布

方案 3：API 发布（仅部分平台）
支持 API 的平台：
- Twitter/X（通过 Twitter API）
- LinkedIn（通过 LinkedIn API）
- Medium（通过 Medium API）
- Dev.to（通过 Dev.to API）

不支持的中文平台：
- ❌ 公众号（需要服务号）
- ❌ 小红书
- ❌ 知乎
- ⚠️ 微博（API 限制严格）
```
