# Interaction Presets

这是 `ui-polish` 给常见交互诉求准备的轻量 preset。目标不是让用户补实现细节，而是让 agent 在收到自然语言描述后，先把行为意图收束成默认实现方向和 review 重点。

## 当前预设

- `hover-explain`
  - 适合“停留悬浮显示说明”“hover 提示”“鼠标停留显示补充信息”这类诉求
  - 默认解释为透明、轻量、小尺寸 tooltip / 小浮层，不是大弹窗
  - 优先图标、logo、短标签，不要长句

- `click-feedback`
  - 适合“按钮点击后要立刻反馈”“点了要知道有没有生效”这类诉求
  - 默认解释为按下态 + loading / disabled + 结果反馈

- `drag-reorder-grid`
  - 适合“按住拖动换位置”“像手机图标一样换位”“拖动排序”这类诉求
  - 默认解释为同级网格重排，不是自由拖拽布局

## 使用原则

- 用户已经用自然语言说清交互目标时，先命中 preset，再补实现。
- 不要反过来要求用户先给出“sortable grid”“pointer capture”这类术语。
- preset 只给默认边界，不代替具体实现。
