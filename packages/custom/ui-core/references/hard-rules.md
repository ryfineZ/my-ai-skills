# Hard Rules

## 起手

- 先定平台、界面类型、主任务、视觉方向，再开始实现。
- 一个界面只保一个主方向，不要混成“什么都有一点”。

## 设计令牌

- 颜色、间距、圆角、边框、阴影、字号层级、动效节奏都必须走 token。
- 组件消费语义 token，不到处散落原始值。

## 文案与配色禁区

- 文案站在使用者视角；不要自我解释、抽象空话、提示词残留、没信息价值的大标题、重复块和冗余解释。
- 默认不写区域标题、眉标题、解释型标题。
- 能靠内容本身、位置关系、状态变化看懂的，就不要标题。
- 没有明确品牌或设计系统依据时，不要默认回到蓝紫/紫粉、蓝紫玻璃、紫白 SaaS、黑底霓虹青、祖母绿/翡翠绿、银灰、雾绿、橙红/焦橙渐变等 AI 惯用安全配色。
- 已被明确否定的组合直接拉黑，不要回滚；当前拉黑项包含：浅暖奶油底 + 棕金玻璃 + 日照穹顶 / 奶油雾面背景。

## 结构与空间

- 尺寸匹配内容；不要把输入框、搜索框、按钮、卡片故意做很大来冒充高级。
- 先判断界面本质是“列表/表单/面板”还是“路径/关系/战术图/时间轨道”；如果本质是后者，默认先用节点、连线、轨道、锚点、贴边参数，不要先落成一堆卡片。
- 图形主屏里的节点、hover 线、数据浮层和当前态必须锚到真实路径、真实图形和真实缩放结果，不能靠肉眼估一个差不多的位置。
- 不是所有元素都要被框起来；只有当边界本身提供了信息分组、操作归属或状态反馈时，才允许做完整容器。
- 间距只够分组；同组贴近，不同组才拉开，标题/工具条/按钮要贴着它控制的内容。
- 间距默认分三档：强关联 `4-8px`，同区并列 `8-12px`，跨区块 `12-16px`；不要全页一个固定间距跑到底。
- 横向空间足够时，不把切换器、筛选器、段控件、标题或摘要硬挤成竖排或窄列。
- 不要嵌没有意义的外框、内框、卡片套卡片；隐藏态也不能留下空壳和占位框。
- 图形层、节点层、图表层不能盖住浮层、按钮、hover 和拖动这些交互层。
- 一组并列浮层如果被当成同一级信息窗，就要一起看顶线、底线、高度和视觉重量，不要一左一右各长各短像临时拼上去。
- 局部去框后必须回看全局；如果为了去掉一个框，换来大片失控留白、重心漂移或路径断掉，视为未完成。
- 主任务必须一眼成立，装饰层不能压过信息层。

## 透明材质与主题

- 透明感要靠底景反差成立：先有可识别的场景层，再有玻璃层；不要只做同色半透明蒙版。
- 玻璃层不得透出硬矩形、直角亮块、裁切断层或背景分块；出现时先查底景亮区、backdrop 裁切和高光层。
- 主题切换默认同时处理场景层、玻璃层、前景层，不是一改全改地偏色。
- 如果目标是“看出不同主题”，默认理解为背景场景 + 配色关系一起变化。
- 玻璃里的信息区不要走两个极端：不要再套小卡片，也不要把文字裸丢在角落；优先做成大玻璃里的弱承载信息窗，靠对齐、比例、字重、局部反射成立。
- 玻璃信息区如果出现多余底框、细线、顶线、占位反射，而这些元素本身没有提供清晰层级，就直接删掉。
- 如果提供透明度控制，数值语义必须真实；`0%` 就是整块透明，而且不能留下看不见但会挡操作的透明壳子。

## 可读性、状态、动效

- 图形主屏里的文字要有预算；节点名、状态、参数只保留关键字，不把图形区做成段落阅读区。
- 小字、辅助字、半透明层文字不能发灰发虚；材质、渐变、反射都不能影响正文和状态可读性。
- 换主题后，logo、图标、按钮图形、边线和文字都要重查前景对比，不能陷进背景。
- 时间、金额、计数等稀疏信息先看视觉重量；“数字 + 单位”默认数字做主视觉，单位降到标签或次级层。
- 对齐先看视觉边界，不看代码边界；字形外扩导致看起来没对齐时，要做视觉补偿。
- 同层只保留必要操作；重复、近义、同权重按钮必须合并或删掉。
- 必须补齐 `hover`、`active`、`focus`、`disabled`、`loading`、`empty`、`error`。
- 动效只服务反馈和层级切换，优先 `transform` / `opacity`，不要堆无意义特效。
- 先把位置、大小、字体气质做顺眼，再谈“液晶感”“科技感”“材质感”；排版和比例不对时，材质越多越难看。
- 按钮不能只是半透明胶囊；要有清楚的结构和受力反馈，至少要看出边缘高光、按下收缩、处理中和完成态的区别。
- 日志带、终端带、数据带这类“内容主导区”，标题必须让位于内容本身；标题做成低权重标签，不要做成抢戏大栏头。
