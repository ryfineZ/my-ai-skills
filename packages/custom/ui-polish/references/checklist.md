# Final Checklist

- 是否先定义了平台、界面类型和视觉方向？
- 是否优先复用了现有组件库和设计系统？
- 是否统一使用 token，而不是散落原始值？
- 是否有清晰的主次层级？
- 文案是否站在使用者视角，且已经去掉抽象空话、提示词残留、冗余标题和重复块？
- 是否默认删掉了区域标题、眉标题、解释型标题？
- 那些标题如果删掉，界面是否仍然完全成立？如果是，就继续删。
- 是否补齐关键状态：hover、focus、disabled、loading、empty、error？
- 尺寸和间距是否服从内容关系，没有故意做大、做远、做空？
- 当前界面如果本质是路径图、战术图、关系图或时间轨道，是否真的使用了节点、连线、轨道、锚点，而不是换皮卡片？
- 如果本质是图形主屏，当前节点、后续节点、完成节点是否都真正贴在线路或轨道上，而不是视觉上看着差不多？
- 是否克制了“什么都加框”的冲动，只有必须分组/反馈的地方才用了完整边界？
- 是否去掉了没有信息价值的外框/内框，以及隐藏态留下的空壳容器？
- 同组标题、工具条、按钮和内容是否贴合？横向空间足够时，控件是否优先横排？
- 间距是否按强关联、同区并列、跨区块分级，而不是所有地方都一个固定值？
- 大留白区里的标题、摘要和角标文案是否避免了不必要的强制换行或窄列排版？
- 去掉某个容器后，是否回看了全局重心和留白，没有留下新的大片空白死区？
- 小号文字、辅助文字和玻璃层文字是否仍然清晰可读？
- 图形主屏里的文字是否已经压缩到关键字，而不是仍然像卡片摘要或说明文？
- 如果用了透明/玻璃层，是否先有可识别的场景层，再有玻璃层？
- 玻璃层里是否出现了直角亮块、背景分块、裁切断层或类似的矩形伪影？
- 如果有玻璃里的信息区，它是否是“弱承载信息窗”而不是卡片套卡片，也不是没有承载的裸字？
- 玻璃信息区里是否还残留多余底框、细线、顶线、装饰反射？
- 图形层、节点层、图表层是否没有盖住浮层、按钮、hover、拖动这些交互层？
- 如果有一组对称或并列浮层，它们的顶线、底线、高度和视觉重量是否协调，而不是左高右矮、左重右轻？
- 如果浮层支持拖动或透明度调节，是否真的可用、可记住，并且 `0%` 真的是整块透明？
- 如果要看出主题差异，是否连同背景场景和前景对比一起变化，而不是只改若干色值？
- 时间、金额、数字统计等稀疏信息是否按视觉重量和视觉边界排过，而不是只按字符串长度或代码右对齐排？
- “数字 + 单位”是否避免了单位和数字抢同一层级？
- 如果用了图表 hover，竖线、节点和数据浮层是否贴着真实图形，而不是凭估算悬空？
- 日志带、终端带、数据带等内容主导区，标题是否已经降权，没有盖过内容本体？
- 是否只有少量且有目的的动效？
- 位置、大小、字体气质是否已经顺眼，再去追液晶感、科技感、材质感？
- 按钮是否不只是“半透明胶囊”，而是有清楚结构、按下反馈、处理中和完成态？
- 是否符合该平台的使用习惯，而不是机械套网页模板？
- 如果用了 Tahoe / Liquid Glass，玻璃是否只服务导航、分层和控制，而没有压过主内容？
- 是否避开了已被否定的背景/配色组合，而不是换个透明度继续复用？
- 是否看起来像真实产品，而不是只把内容摆上去？
