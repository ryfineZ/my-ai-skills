# Final Checklist

- 是否先定义了平台、界面类型和视觉方向？
- 是否优先复用了现有组件库和设计系统？
- 是否统一使用 token，而不是散落原始值？
- 是否有清晰的主次层级？
- 是否有足够留白和分组感？
- 文案是否站在使用者视角，而不是系统解释自己？
- 是否去掉了 AI 味重的抽象句、提示词残留词组和没信息价值的大标题？
- 是否删掉了没必要的眉标题、分组小标题和重复提示？
- 是否合并了重复块，删除了冗余标题和解释？
- 是否补齐关键状态：hover、focus、disabled、loading、empty、error？
- 组件尺寸是否匹配内容和文案长度，而不是空占空间？
- 是否去掉了没有信息价值的外框/内框？
- 组件间距是否足够分组但不过大，没有故意拉空？
- 小号文字、辅助文字和玻璃层文字是否仍然清晰可读，没有故意做淡？
- 是否只有少量且有目的的动效？
- 是否符合该平台的使用习惯，而不是机械套网页模板？
- 如果用了 Tahoe / Liquid Glass，玻璃是否只服务导航、分层和控制，而没有压过主内容？
- 是否看起来像真实产品，而不是只把内容摆上去？
