# Common Patterns

## Popup

- 顶部：标题 + 单个主动作
- 中部：状态摘要 / 关键操作
- 底部：次要入口或帮助
- 控制高度，避免让用户在小窗里深滚动

## Settings

- 左侧分组，右侧详情，或单栏分区式布局
- 先展示最常改的选项
- 危险操作独立分组并弱化常态权重

## Dashboard

- 顶部先给总览与筛选
- 中部按任务流排布，而不是平均切豆腐块
- 卡片不要全等，重要信息要更大、更近、更亮

## Side Panel

- 适合渐进式任务，不适合塞完整应用
- 顶部固定标题与关闭/返回
- 中段主内容可滚动，底部固定主动作

## Command Surface

- 输入区永远是视觉焦点
- 结果项强调选中态和键盘导航
- 装饰尽量少，反馈要快

## Mobile Home

- 顶部只保留核心标题与少量关键动作
- 首屏优先展示当前任务最相关的信息
- 卡片和区块不要过多同权，主路径要一眼可见

## Detail / Form

- 信息先分组再展开，不要把所有字段平铺到底
- 表单错误、说明、成功反馈尽量贴近字段和动作
- 底部主操作区要稳定、清晰、易点击
