# Common Patterns

## Popup

- 顶部：标题 + 单个主动作
- 中部：状态摘要 / 关键操作
- 底部：次要入口或帮助
- 控制高度，避免让用户在小窗里深滚动

## Settings

- 左侧分组，右侧详情，或单栏分区式布局
- 先展示最常改的选项
- 危险操作独立分组并弱化常态权重

## Dashboard

- 顶部先给总览与筛选
- 中部按任务流排布，而不是平均切豆腐块
- 卡片不要全等，重要信息要更大、更近、更亮

## Side Panel

- 适合渐进式任务，不适合塞完整应用
- 顶部固定标题与关闭/返回
- 中段主内容可滚动，底部固定主动作

## Command Surface

- 输入区永远是视觉焦点
- 结果项强调选中态和键盘导航
- 装饰尽量少，反馈要快

## Graph Surface

- 本质是路径图、战术图、关系图、时间轨道时，先搭节点、连线、轨道、锚点，再补信息层
- 当前节点也必须在线上，只比普通节点重一档，不要单独做成脱离线路的大主角卡
- 参数、状态、告警优先做贴边轻浮层，不要回退成大信息卡
- 浮层默认靠四周，不往主路径中间挤；需要避让时，优先支持拖动和记住位置
- 图形层不能挡住浮层按钮、hover 和拖动操作
- 点位、hover 线、节点、图表浮层都要贴真实图形，不要用估算位置硬摆
- 如果提供透明度控制，`0%` 就应该是整块透明，而且不能留下看不见但会挡点击的空壳

## Mobile Home

- 顶部只保留核心标题与少量关键动作
- 首屏优先展示当前任务最相关的信息
- 卡片和区块不要过多同权，主路径要一眼可见

## Detail / Form

- 信息先分组再展开，不要把所有字段平铺到底
- 表单错误、说明、成功反馈尽量贴近字段和动作
- 底部主操作区要稳定、清晰、易点击
