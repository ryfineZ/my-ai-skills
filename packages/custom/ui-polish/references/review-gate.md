# Review Gate

## Rule

做完界面后，不允许直接以“好看”或“完成”收尾。

必须执行下面闭环：

1. 生成真实截图
2. 对照本清单逐项打分
3. 出现任一阻断项就必须继续修改
4. 只有截图和清单都过关，才能交付

## Blocking Issues

任一命中都算不通过：

- 主任务入口权重失衡
- 标题、卡片、按钮或输入区尺寸明显失衡
- 文案像系统在解释自己，或存在抽象空话、提示词残留、没信息价值的大标题
- 重复块、重复标题、冗余小标题、眉标题、分组标题没有删除
- 本可删除的区域标题、眉标题、解释型标题仍然占位、抢视线
- 在没有明确品牌依据时，仍然默认落回蓝紫/紫粉渐变、蓝紫玻璃、紫白 SaaS、黑底霓虹青、祖母绿/翡翠绿、银灰、雾绿、橙红/焦橙渐变等常见 AI 惯用配色
- 配色没有明确主色关系，或一眼就是通用模板味
- 字体气质与产品类型不匹配
- 小号文字、辅助文字或玻璃层文字对比度过低，出现发虚、发灰、读不清
- 信息层级混乱，或间距、尺寸、容器明显空耗空间
- 界面该做成路径图/关系图/战术图/时间轨道，却仍然用一堆卡片、列表或信息框表达
- 图形主屏里的节点、当前态、hover 线或数据浮层没有贴真实路径和真实图形，出现点在线外、线和点错位、跟缩放脱节
- 大留白里的标题、摘要或角标文案被不必要地硬换行，右侧还有空间却被挤成窄列
- 嵌套了没有信息价值的外框、内框，容器层级过多
- 强关联、同区并列、跨区块没有间距分级，全页像一个固定 gap 模板
- 局部去框后留下新的大面积空白、重心漂移或路径断裂
- 玻璃层里出现直角亮块、背景分块、裁切断层或其他矩形伪影
- 稀疏信息区的时间、金额、计数等看起来没有视觉对齐，或“数字 + 单位”同级争抢主视觉
- 按钮点下后没有明确反馈，用户无法判断是否生效或是否处理中
- 同层按钮重复、近义、同权重，主次不清
- 状态缺失或像静态海报
- 图层堆叠关系错了，图形层、节点层或透明壳子挡住浮层、按钮、hover、拖动等实际交互
- 成对或同组浮层的顶线、底线、高度、位置关系明显失衡，像临时摆上去的两块东西
- 浮层透明度语义不真实；例如 `0%` 仍然能看见内容，或虽然看不见却还挡操作
- 图形主屏的文字过多，把节点区做成段落阅读区
- 日志带、终端带、数据带等内容主导区，标题或栏头比内容更抢眼
- 为了“质感”牺牲可读性和操作效率
- 明显像 AI 默认模板拼出来的页面

## Scoring

每项按 `1-5` 分：

- 层级
- 留白
- 配色
- 字体
- 文案
- 尺寸匹配
- 容器克制
- 间距控制
- 平台适配
- 状态完整度
- 动效克制
- 成品感

任一项低于 `3` 分，不通过。
总分低于 `46/60`，不通过。

## Expected Output

交付时至少回答：

- 这张界面的主任务是什么
- 最重的视觉元素是否真的值得最重
- 哪些标题、解释、按钮或容器其实应该删掉或合并
- 为什么这套配色不是模板色
- 字体、对比度和尺寸为什么够读、够用
- 哪些按钮反馈被补齐了
- 哪些地方被主动克制了
