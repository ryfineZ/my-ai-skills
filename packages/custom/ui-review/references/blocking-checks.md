# Blocking Checks

任一命中都算不通过：

- 主任务入口不成立，或主次权重明显失衡。
- 文案像系统在解释自己，或仍有抽象空话、提示词残留、没信息价值的大标题、重复块、冗余小标题。
- 区域标题、眉标题、解释型标题本来可以删，却还留着占注意力。
- 在没有明确品牌依据时，仍默认回到蓝紫/紫粉、蓝紫玻璃、紫白 SaaS、黑底霓虹青、祖母绿/翡翠绿、银灰、雾绿、橙红/焦橙渐变等 AI 惯用配色，或整体一眼就是模板味。
- 字体气质不匹配产品类型，或小字、辅助字、玻璃层文字对比度过低，发灰发虚读不清。
- 小字/辅助字如果用了弱色 token、透明度、模糊层或玻璃层叠加，运行时一旦出现局部看不清，直接阻断。
- logo、图标、按钮图形、边线或文字在某个主题下陷进背景，导致看不到、看不清。
- 标题、卡片、按钮、输入区尺寸失衡，或间距、容器、层级明显空耗空间。
- 界面本质明明是路径图、关系图、战术图、时间轨道，却落成了一堆卡片、列表块或信息框，表达方式和任务隐喻不匹配。
- 图形主屏里的节点、当前态、hover 线或数据浮层没有贴真实路径和真实图形，出现点在线外、线和点错位、跟缩放脱节。
- 多栏并排时，外边距、顶底留白、左右栏高度或主面板底线明显不齐，导致版心和边界关系失衡。
- 同组内容被拉远，标题/工具条/按钮与它控制的内容脱节；横向空间足够却硬做竖排或窄列，制造无意义留白。
- 全页或主屏使用单一固定间距，强关联、同区并列、跨区块没有分级，导致空间关系发傻。
- 去掉局部边框后留下大片空白死区、重心漂移或路径断裂，只解决局部没有解决全局。
- 时间、金额、计数等稀疏信息虽然“代码上对齐”，但视觉边界没对齐，或“数字 + 单位”同级抢主视觉，导致角标区发笨发挤。
- 透明层和底层几乎同色，玻璃只像半透明蒙版；主题切换还是一改全改，看不出场景层和玻璃层的关系变化。
- 玻璃层透出了硬矩形、直角亮块、背景分块、裁切断层或类似伪影。
- 页面留下空白壳子、透明占位框、功能不明的隐藏容器，或嵌了没有信息价值的外框/内框。
- 图层堆叠关系错了，图形层、节点层或透明壳子挡住浮层、按钮、hover、拖动等实际交互。
- 成对或同组浮层的顶线、底线、高度、位置关系明显失衡，像临时摆上去的两块东西。
- 浮层透明度语义不真实；例如 `0%` 仍然能看见内容，或虽然看不见却还挡操作。
- 图形主屏里的文案过多，节点和路径区变成段落阅读区，图形表达被文字吞掉。
- 日志带、终端带、数据带的标题或栏头比内容本体还重，形成喧宾夺主。
- 按钮点下后没有明确反馈，或同层按钮重复、近义、同权重，主次不清。
- 状态缺失，页面更像静态海报而不是可操作界面。
- 图表 hover 的竖线、节点、数据浮层没贴真实图形，出现悬空、错位或估算点。
- 为了“质感”牺牲可读性和操作效率。
